# Theory

## Beamforming in Context

Beamforming uses an array of spatially separated sensors or radiators to control how signals add in space. By applying complex weights across the array, the system can reinforce energy from a desired direction while suppressing energy from other directions.

In the narrowband receive model, the beamformer output is

```math
y[k] = \mathbf{w}^H \mathbf{x}[k]
```

where:

- `x[k]` is the complex array snapshot at time index `k`
- `w` is the complex weight vector
- `(\cdot)^H` denotes conjugate transpose

The same weighting structure also appears in transmit-array pattern synthesis, where one studies the array factor generated by a chosen set of complex excitations.

## Geometry and Steering Intuition

For a uniform linear array (ULA) along the x-axis, adjacent elements are spaced by `d`. In this repository, spacing is normalized by wavelength:

```math
spacing\_lambda = \frac{d}{\lambda}
```

Under a far-field plane-wave assumption, the relative phase across the aperture is linear. A steering vector therefore applies a phase ramp that aligns the array for a desired direction.

Two consequences matter immediately:

- Larger apertures create narrower main lobes.
- Larger spacing increases the risk of grating lobes.

For broadside-style scans in a ULA, `d <= lambda/2` is the standard safe regime for avoiding visible-space grating lobes across typical scans. The tests in this repository explicitly demonstrate that spacing above `lambda/2` can create additional unit-strength lobes.

## Main-Lobe, Sidelobe, and Taper Tradeoffs

Uniform weighting gives the narrowest main lobe for a fixed aperture, but it also produces relatively high sidelobes. Applying an amplitude taper such as Hamming or Taylor changes that tradeoff:

- lower sidelobes
- wider main lobe
- reduced peak directivity compared with uniform weighting

This is a standard aperture-weighting tradeoff. The toolkit exposes these tapers through `core.beamforming.amplitude_taper(...)` and uses them in the pattern and steering helpers.

## Far Field Versus Near Field

The base array-factor functions use a far-field model:

- incoming or outgoing waves are approximated as plane waves
- the phase law is linear across the array
- only direction matters, not finite source range

The near-field helpers in `core.advanced_models` replace that with a distance-based phase law. Instead of steering to a direction only, the array focuses toward a finite-range point:

```math
r_n = \lVert \mathbf{p}_{focus} - \mathbf{p}_n \rVert
```

and the compensating phase depends on `r_n`, not just on a direction cosine. This matters when the wavefront curvature across the aperture is not negligible.

## Narrowband Versus Wideband

The steering laws in classical phased arrays are exact only at the design frequency. If fixed phase-shifter weights are designed at `f_0` and evaluated at another frequency `f`, the electrical spacing changes relative to wavelength. The result is beam squint: the apparent steering direction shifts with frequency.

The repository models this explicitly in `wideband_array_factor_linear(...)` by holding the phase-shifter weights fixed at the center frequency and sweeping the response over multiple frequencies.

## Array Imperfections

Ideal array-factor analysis assumes:

- identical isotropic elements
- no mutual coupling
- exact amplitude and phase control

Real arrays violate all three assumptions. The current toolkit includes simplified models for:

- element patterns through `element_pattern_gain(...)`
- mutual coupling through a coupling matrix
- architecture constraints through digital, analog, and hybrid weight synthesis

These are intentionally simple but useful for understanding first-order deviations from the ideal response.

## Adaptive Beamforming

Conventional steering uses only geometry and the desired look direction. Adaptive beamforming additionally uses data statistics, usually through the spatial covariance matrix:

```math
\mathbf{R} = \mathbb{E}\{\mathbf{x}[k]\mathbf{x}^H[k]\}
```

In practice the expectation is replaced by a sample average over snapshots. Adaptive methods such as MVDR aim to preserve unit gain in the desired direction while minimizing output power from interference and noise.

MUSIC uses the same covariance matrix differently: it separates signal and noise subspaces and then searches for directions whose steering vectors are orthogonal to the estimated noise subspace.

## Practical Defaults Used in This Repo

For exploratory simulations, the following defaults are sensible and consistent with the test suite and dashboard:

- `spacing_lambda = 0.5`
- a ULA unless planar behavior is specifically needed
- uniform taper for aperture intuition, Hamming for lower sidelobes
- far-field modeling unless focusing at a known finite range
- diagonal loading when working with estimated covariance matrices
